Week 4 notes

Topics of the week: forms, form validation, OOP and more!

Forms

Forms are made up of a <form> element that contains form controls such as input fields, select menus and buttons. These input fields can be populated with information that is processed once the form has been submitted.

Form Events

focus: occurs when an element is focused on.

blur: occurs when the user moves the focus away from the form element.

change: occurs when the user moves the focus away from the form element after changing it.

submit: one of the most important ones. It catches the form before it is submitted.

Form Controls

Some common types of form control are:

  • <input> fields, including text, passwords, check boxes, radio buttons, and file uploads
  • <select> menus for drop-down lists of options
  • <textarea> elements for longer text entry
  • <button> elements for submitting and resetting forms

Input fields

These are the most common type of form controls, but they have several categories:

Text: used for entering a short piece of text, such as a username.

Password: it works the same as the text types, except the characters are concealed as they are entered so they're unable to be read on the screen.

Checkbox: used to select different options that can be checked (true) or left unchecked (false).

Radio Button: they allow users to check an option as true, but they provide an exclusive choice of options, so only one option can be selected.

Hidden: are not displayed by the browser, but have a 'value' attribute that can contain information that is submitted with the form. They are often used to send information such as settings or information that the user has already provided.

File: used to upload files, and most browsers will provide a browse button or similar that lets users select a file from their file system.

Others: number, tel, color, etc.

Select Drop-Down List

They can be used to select one or more options from a list of values.

Text Areas

They are used to enter long pieces of text over multiple lines such as a comment or blog post. They work in much the same way as input fields.

Form Validation

Form validation is the process of checking whether a user has entered the information into a form correctly.
